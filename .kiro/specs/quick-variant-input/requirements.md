# 需求文件

## 簡介

本功能為商品管理新增「快速規格輸入」文字欄位，允許使用者以簡寫語法一次定義多組規格維度及其選項值。系統解析輸入後產生對應的 `SpecDimension[]`，再透過既有的笛卡爾積邏輯自動產生所有規格組合（ProductVariant）。此功能旨在大幅減少手動逐一新增規格維度與選項值的操作步驟，提升商品建立效率。

## 術語表

- **Quick_Variant_Input**：快速規格輸入元件，接受使用者輸入的簡寫語法文字並解析為規格維度。
- **Parser**：解析器，負責將簡寫語法字串轉換為結構化的 `SpecDimension[]` 資料。
- **Shorthand_Syntax**：簡寫語法，格式為 `[值1，值2/值3，值4，值5]`，以 `/` 分隔不同維度群組，以全形逗號 `，` 或半形逗號 `,` 分隔同一維度內的選項值，整體以方括號 `[]` 包裹。
- **Dimension_Group**：維度群組，簡寫語法中以 `/` 分隔的一組選項值，對應一個 `SpecDimension`。
- **SpecDimension**：規格維度資料結構，包含維度名稱（`name`）與選項值列表（`values`）。
- **Cartesian_Product**：笛卡爾積，將所有維度的選項值進行全排列組合，產生所有可能的規格組合。
- **Pretty_Printer**：格式化輸出器，將結構化的 `SpecDimension[]` 轉換回簡寫語法字串。

## 需求

### 需求 1：解析簡寫語法

**使用者故事：** 身為商品管理員，我希望能輸入簡寫語法文字來定義規格維度，以便快速建立多維度規格組合而不需逐一手動新增。

#### 驗收條件

1. 當提供有效的簡寫語法字串時，解析器應將其解析為 `SpecDimension[]` 陣列，其中以 `/` 分隔的每個維度群組成為一個 `SpecDimension` 項目。
2. 當簡寫語法中同一維度群組內的值以全形逗號 `，` 或半形逗號 `,` 分隔時，解析器應將每個分隔值視為該維度 `values` 陣列中的一個選項。
3. 當簡寫語法以方括號 `[` 和 `]` 包裹時，解析器應去除括號後再解析內部內容。
4. 當簡寫語法未使用方括號時，解析器應直接解析內容，不要求必須有括號。
5. 解析器應去除每個解析出的選項值前後的空白字元。
6. 解析器應移除同一維度群組內重複的選項值，保留首次出現的順序。
7. 解析器應依維度群組的順序自動產生維度名稱，格式為 `維度1`、`維度2`、`維度3` 等。
8. 當簡寫語法僅包含一個維度群組且無 `/` 分隔符時，解析器應產生單一個名為 `維度1` 的 `SpecDimension`。

### 需求 2：處理無效輸入

**使用者故事：** 身為商品管理員，我希望在輸入格式錯誤時收到明確的錯誤提示，以便修正輸入並成功產生規格組合。

#### 驗收條件

1. 當簡寫語法字串為空或僅包含空白字元時，解析器應回傳空的 `SpecDimension[]` 陣列，不產生錯誤。
2. 當某個維度群組不包含有效的選項值（僅有分隔符或空白）時，解析器應從結果中排除該空維度群組。
3. 當所有維度群組解析後皆為空時，解析器應回傳空的 `SpecDimension[]` 陣列，不產生錯誤。
4. 若解析結果將產生超過 100 個規格組合（笛卡爾積超過 100），快速規格輸入元件應顯示警告訊息，說明組合數量及 100 組合上限。

### 需求 3：格式化輸出規格維度

**使用者故事：** 身為商品管理員，我希望看到結構化的規格維度以簡寫語法格式顯示，以便確認解析結果是否正確。

#### 驗收條件

1. 格式化輸出器應將 `SpecDimension[]` 陣列格式化為有效的簡寫語法字串，使用 `/` 作為維度分隔符，`，` 作為同一維度內的值分隔符，並以 `[]` 包裹。
2. 對所有有效的 `SpecDimension[]` 陣列，解析格式化後的字串應產生等價的 `SpecDimension[]`（往返屬性：parse(print(x)) 產生等價的 values 陣列）。

### 需求 4：快速規格輸入 UI 元件

**使用者故事：** 身為商品管理員，我希望在商品新增/編輯頁面有一個快速輸入欄位，以便用簡寫語法快速定義規格維度。

#### 驗收條件

1. 快速規格輸入元件應渲染一個文字輸入欄位，佔位文字為 `例：白，黑/35，36，37，38，39，40`。
2. 快速規格輸入元件應在文字輸入欄位上方顯示標籤 `快速規格輸入`。
3. 當使用者在輸入欄位中輸入或貼上文字時，快速規格輸入元件應即時解析輸入並在輸入欄位下方顯示已解析維度群組的預覽。
4. 當解析結果有效且非空時，快速規格輸入元件應顯示「套用」按鈕，觸發產生 `SpecDimension[]` 並傳遞給父元件。
5. 當使用者點擊「套用」按鈕時，快速規格輸入元件應呼叫父元件的回呼函式，傳入解析後的 `SpecDimension[]`，取代目前的規格維度。
6. 當解析結果會產生規格組合時，快速規格輸入元件應顯示總組合數（例如：`將產生 12 個規格組合`）。
7. 若組合數超過 100，快速規格輸入元件應停用「套用」按鈕並以錯誤色彩顯示警告。

### 需求 5：與商品表單整合

**使用者故事：** 身為商品管理員，我希望快速規格輸入與現有商品表單整合，以便套用後的規格維度能直接反映在商品的規格組合中。

#### 驗收條件

1. 當點擊「套用」按鈕時，快速規格輸入元件應以解析後的維度更新商品的 `specDimensions` 欄位。
2. 當透過快速規格輸入更新 `specDimensions` 時，商品表單應觸發既有的笛卡爾積產生邏輯，產生新的 `ProductVariant[]` 項目。
3. 當商品已有包含庫存或價格覆寫的既有規格組合時，快速規格輸入元件應顯示確認對話框，警告套用新維度將取代所有既有規格組合。
4. 當商品表單處於唯讀或檢視模式時，快速規格輸入元件應隱藏。

### 需求 6：解析器往返正確性

**使用者故事：** 身為開發者，我希望確保解析器與格式化器的往返正確性，以便資料在轉換過程中不會遺失。

#### 驗收條件

1. 對所有產生非空結果的有效簡寫語法字串，格式化後再解析應產生相同的 `SpecDimension[]` values 陣列（往返：parse(print(parse(input))) 等於 parse(input)）。
2. 對所有包含非空維度與非空值的有效 `SpecDimension[]` 陣列，parse(print(dims)) 應產生與原始 dims 等價的 values 陣列。
3. 解析器應為無副作用的純函式，接受字串並回傳 `SpecDimension[]` 陣列。
4. 格式化輸出器應為無副作用的純函式，接受 `SpecDimension[]` 陣列並回傳字串。
