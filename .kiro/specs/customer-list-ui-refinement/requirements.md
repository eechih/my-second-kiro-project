# 需求文件

## 簡介

本規格定義客戶列表頁面的 UI 佈局重構需求，以 Mantis Dashboard 的客戶列表頁面為參考設計，將現有的客戶管理列表頁面升級為更專業、功能更豐富的表格介面。重構範圍包含工具列佈局、表格欄位結構、行操作按鈕、狀態顯示方式、批次選取功能及分頁元件。

## 術語表

- **Customer_List_Page**: 客戶列表頁面元件，位於 `src/routes/customers/index.tsx`，負責顯示客戶資料表格及相關操作
- **Toolbar**: 頁面頂部的工具列區域，包含搜尋框、篩選下拉選單、排序下拉選單、新增按鈕及匯出按鈕
- **DataTable**: 共用表格元件，位於 `src/components/DataTable.tsx`，使用 TanStack Table + MUI Table 渲染
- **User_Info_Cell**: 表格中合併顯示客戶名稱與聯絡人資訊的複合儲存格
- **Status_Text**: 以彩色文字（非 Chip）顯示客戶啟用狀態的呈現方式
- **Row_Actions**: 每列資料末端的操作圖示按鈕群組（檢視、編輯、停用/啟用、刪除）
- **Bulk_Selection**: 透過核取方塊選取多列資料以進行批次操作的功能
- **Pagination_Control**: 表格底部的分頁控制元件，採用游標式分頁（基於 DynamoDB nextToken），包含每頁筆數選擇及上一頁/下一頁導覽按鈕
- **Status_Filter**: 工具列中用於篩選客戶啟用狀態的下拉選單元件
- **Sort_Dropdown**: 工具列中用於選擇排序欄位的下拉選單元件
- **CSV_Export**: 將目前篩選結果匯出為 CSV 檔案的功能

## 需求

### 需求 1：工具列佈局重構

**使用者故事：** 身為系統操作人員，我希望工具列將搜尋、篩選、排序、新增及匯出功能整合在同一列，以便快速存取所有列表操作而不需要在多個區域之間切換。

#### 驗收條件

1. Customer_List_Page 應顯示單列式 Toolbar，由左至右水平排列搜尋輸入框、Status_Filter 下拉選單、Sort_Dropdown、新增客戶按鈕及 CSV_Export 圖示按鈕。
2. 頁面載入時，Toolbar 的搜尋輸入框應顯示包含總筆數的佔位文字，格式為「搜尋 {N} 筆記錄...」。
3. Status_Filter 應提供「全部狀態」、「啟用中」及「已停用」三個選項，以 MUI Select 下拉選單元件取代現有的 ToggleButtonGroup。
4. Sort_Dropdown 應提供排序選項，包含「客戶名稱」、「聯絡人」、「電話」及「建立日期」，以 MUI Select 下拉選單元件呈現。
5. 當 Sort_Dropdown 的值變更時，DataTable 應依所選欄位重新排序顯示的資料。
6. 新增客戶按鈕應顯示「新增客戶」文字，使用 primary 色彩變體並帶有加號圖示。
7. CSV_Export 按鈕應以 IconButton 呈現，使用下載/匯出圖示，位於新增客戶按鈕之後。

### 需求 2：客戶資訊複合欄位

**使用者故事：** 身為系統操作人員，我希望客戶名稱與聯絡人資訊合併顯示在同一欄位中，以便在有限的表格空間內快速辨識客戶身份。

#### 驗收條件

1. DataTable 應顯示一個「客戶資訊」欄位，將客戶名稱與聯絡人合併於同一個表格儲存格中。
2. User_Info_Cell 應將客戶名稱以粗體文字（Typography variant subtitle2）顯示於第一行，聯絡人名稱以次要色彩文字（Typography variant body2）顯示於第二行。
3. User_Info_Cell 應在文字內容左側顯示圓形 Avatar 元件，顯示客戶名稱的第一個字元。
4. Avatar 元件應使用由客戶名稱字串衍生的一致背景色彩。

### 需求 3：表格欄位結構

**使用者故事：** 身為系統操作人員，我希望表格欄位結構經過重新設計，以便在一個畫面中看到最重要的客戶資訊。

#### 驗收條件

1. DataTable 應依以下順序顯示欄位：Bulk_Selection 核取方塊、列號（#）、User_Info_Cell（客戶資訊）、電話、Email、地址、Status_Text（狀態）及 Row_Actions（操作）。
2. 列號欄位應顯示從 1 開始的連續編號，依據目前頁碼與每頁筆數計算（page × pageSize + rowIndex + 1）。
3. Status_Text 應以綠色（success.main）顯示「啟用中」代表啟用客戶，以紅色（error.main）顯示「已停用」代表停用客戶。
4. 當欄位標題被點擊時，DataTable 應依該欄位排序資料；核取方塊、列號及操作欄位不可排序。

### 需求 4：行操作按鈕

**使用者故事：** 身為系統操作人員，我希望每列資料提供檢視、編輯及停用/啟用的操作按鈕，以便直接從列表頁面執行常用操作。

#### 驗收條件

1. Row_Actions 欄位應為每列顯示三個圖示按鈕：檢視（眼睛圖示）、編輯（鉛筆圖示）及啟用/停用（切換圖示）。
2. 當檢視按鈕被點擊時，Customer_List_Page 應導覽至客戶詳情頁面，路由為「/customers/$customerId」。
3. 當編輯按鈕被點擊時，Customer_List_Page 應導覽至客戶編輯頁面，路由為「/customers/$customerId」並帶有編輯模式參數。
4. 當停用按鈕被點擊（針對啟用中的客戶）時，Customer_List_Page 應顯示確認對話框後再執行停用操作。
5. 當啟用按鈕被點擊（針對已停用的客戶）時，Customer_List_Page 應顯示確認對話框後再執行啟用操作。
6. 啟用/停用按鈕應為啟用中的客戶顯示禁止圖示（warning 色彩），為已停用的客戶顯示勾選圖示（success 色彩）。

### 需求 5：批次選取功能

**使用者故事：** 身為系統操作人員，我希望能透過核取方塊選取多筆客戶資料，以便未來支援批次操作（如批次停用）。

#### 驗收條件

1. DataTable 應在標題列顯示一個核取方塊，點擊時可選取或取消選取所有可見列。
2. DataTable 應在每個資料列顯示一個核取方塊，點擊時可選取或取消選取該列。
3. 當一列或多列被選取時，Customer_List_Page 應以醒目背景色彩標示已選取列的視覺狀態。
4. 當標題列核取方塊處於不確定狀態（部分列被選取）時，核取方塊應顯示不確定狀態的視覺指示。

### 需求 6：分頁元件（游標式分頁）

**使用者故事：** 身為系統操作人員，我希望分頁元件提供上一頁/下一頁的導覽功能，以便在資料列表中逐頁瀏覽客戶資料。

#### 驗收條件

1. Pagination_Control 應顯示「每頁筆數」下拉選單，提供 10、25 及 50 的選項。
2. Pagination_Control 應顯示「下一頁」按鈕；當後端回傳 nextToken 時按鈕為啟用狀態，否則為停用狀態。
3. Pagination_Control 應顯示「上一頁」按鈕；當存在前一頁的 token 歷史記錄時按鈕為啟用狀態，否則為停用狀態。
4. 當使用者點擊「下一頁」時，Customer_List_Page 應使用後端回傳的 nextToken 作為參數請求下一頁資料。
5. 當使用者點擊「上一頁」時，Customer_List_Page 應從本地維護的 token 堆疊中取出前一頁的 token 請求該頁資料。
6. 當使用者變更每頁筆數或篩選條件時，分頁狀態應重置（清除 token 堆疊，從第一頁重新載入）。
7. Pagination_Control 應顯示目前頁面的筆數資訊，格式為「顯示 {count} 筆」。

### 需求 7：CSV 匯出功能

**使用者故事：** 身為系統操作人員，我希望能將目前篩選後的客戶列表匯出為 CSV 檔案，以便在外部工具中進行進一步分析或備份。

#### 驗收條件

1. 當 CSV_Export 按鈕被點擊時，Customer_List_Page 應產生包含所有目前篩選結果的 CSV 檔案。
2. CSV 檔案應包含以下欄位：客戶名稱、聯絡人、電話、Email、地址、狀態、建立日期。
3. CSV 檔案應使用 UTF-8 編碼並帶有 BOM，以確保繁體中文字元在試算表應用程式中正確顯示。
4. CSV 檔案應以「customers\_{YYYY-MM-DD}.csv」格式命名，使用當天日期。
5. 當匯出進行中時，CSV_Export 按鈕應顯示載入指示器並停用，以防止重複匯出。

### 需求 8：頁面標題與麵包屑

**使用者故事：** 身為系統操作人員，我希望頁面頂部顯示麵包屑導覽路徑，以便了解目前所在位置並快速返回上層頁面。

#### 驗收條件

1. Customer_List_Page 應在頁面標題上方顯示麵包屑導覽，呈現「首頁 / 客戶 / 列表」。
2. 當「首頁」麵包屑連結被點擊時，Customer_List_Page 應導覽至儀表板頁面，路由為「/」。
3. Customer_List_Page 應在麵包屑下方以 Typography variant h5 顯示「列表」作為頁面標題。
